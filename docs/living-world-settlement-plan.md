# Earth on Minecraft 生物、聚落与品质升级计划

日期：2026-07-17
目标平台：Minecraft 26.2 / NeoForge 26.2.0.7-beta
计划定位：本体下一条大型主线

## 1. 目标

Earth on Minecraft 已经验证了地质、机器、电力、物流、手册和 JEI 的基本功能。下一阶段不再以“先有一个能工作的方块”为目标，而是把世界变成真正有人生活、生产、交易、建设和防御的现实地球缩影。

这条主线同时解决四个问题：

1. 原版村民只有职业和交易差异，缺少个人身份、社会角色和聚落技术环境。
2. 原版村庄规模小、功能单一，不能承载本体已经存在的矿业、制造、电力、水处理和物流体系。
3. 掠夺者的行为、装备和掉落与目标聚落的发展水平无关。
4. 机器目前主要通过贴图、进度条和少量粒子表达工作，缺少与现实过程一致的动作、声音和成型结构效果。

最终效果不是现代大都市模拟，也不是对着电脑上班的办公社会。重点是村落、集镇、资源镇、工业镇、工厂、矿区、发电站、水务设施、仓储和运输节点。

## 2. 本体与独立项目边界

### 2.1 属于 Earth on Minecraft 本体

- 原版村民、流浪商人和掠夺者的兼容增强。
- 居民姓名、职业角色、技能等级、工作地点和聚落归属。
- 村落、集镇、资源镇和工业镇的生成与发展。
- 矿场、工坊、窑场、工厂、发电站、变电设施、水处理、仓库和货运节点。
- 与聚落环境相关的交易、物资需求、掉落、任务和防御。
- 铁/钢制管道、阀门、泵、储罐、配电箱、路灯和其他实际基础设施。
- 居民工作动作、机器运行效果、环境声音和多方块成型动画。

### 2.2 不进入本体

- 玩家和居民的真实饥饿、疾病、营养、体温、伤口和生活压力，继续由 Earth Human 负责。
- 枪炮与武器工业路线。
- 超大型现代都市、写字楼、办公室职业和完整城市经济模拟。
- 建筑承重、地基、钢筋配置、结构失效、施工安全和规范验算。

最后一项适合未来独立的硬核工程模组。Earth on Minecraft 可以提供水泥、钢材、管道和机械，但不强迫普通玩家先打地基、配钢筋和计算荷载才能建房。

## 3. 不直接把“村民”替换成大量新实体

第一阶段保留原版 `Villager` 作为兼容底座，并在其上增加持久化居民档案。这样可以继续兼容原版职业、繁殖、床、POI、交易、声望、灾厄巡逻和其他 mod 对村民的识别。

只有在模型、行为和生命周期确实完全不同的情况下，才增加新的实体类型。

居民档案至少包含：

- 稳定个人姓名。
- 聚落 ID。
- 社会身份：村民、镇民、工人、技师、商人等。
- 具体职业角色。
- 技能等级或资历。
- 当前工作地点和工作状态。
- 所属聚落的发展阶段与产业方向。
- 可选的外观种子、服饰和安全装备。

玩家看到的名称可以是：

```text
林川 · 矿工
陈岚 · 机修技师
镇民 Nora · 水务操作员
```

个人姓名生成后保持稳定；职业标题随语言设置翻译。姓名池、称谓格式和文化风格使用数据包配置，不在 Java 中写死唯一文化。

原版出生蛋和底层实体仍可保留“村民”名称，但世界内居民应根据聚落规模和角色显示“村民”“镇民”“工人”“技师”等更准确身份。

## 4. 聚落不是一条简单升级线

聚落由三个维度共同决定，而不是所有村庄都按同一路线升级。

### 4.1 规模

- 村落：住宅、农田和少量基础工坊。
- 集镇：市场、仓库、公共设施和多种职业。
- 城镇：多个功能片区、较完整基础设施和工业边缘区。

第一阶段不生成大都市。

### 4.2 技术阶段

- 农业/手工业：保留原版农牧、木工、石工和铁匠体系。
- 机械化：出现矿场、砖窑、锯木、机械维修和小型动力设备。
- 蒸汽/早期工业：出现锅炉房、集中工坊、货运和大型加工设备。
- 电气化工业：出现发电站、变电、照明、电动机器、水处理和自动化前置。

本体复刻内容而不复刻发展痛苦。玩家不需要等待现实年代，也不需要管理完整人口经济，技术阶段主要用于决定环境、建筑、职业和交易内容。

### 4.3 产业专长

- 农业与灌溉。
- 矿业与采石。
- 林业与木材加工。
- 渔业与河港。
- 陶瓷、玻璃和建材。
- 冶金和机械制造。
- 化工与水处理。
- 电力与能源。
- 仓储、市场和货运。

聚落专长优先参考本地生物群系、地形、地质角色和附近资源，再加入受控随机。矿业镇应靠近可解释的矿床，窑场应能获得黏土或燃料，水务设施应与河流、水源或人口规模有关。

## 5. 聚落数据模型

建议建立以下数据源：

```text
data/earth_on_minecraft/settlements/profiles/*.json
data/earth_on_minecraft/settlements/resident_roles/*.json
data/earth_on_minecraft/settlements/name_pools/*.json
data/earth_on_minecraft/settlements/trade_pools/*.json
data/earth_on_minecraft/settlements/building_sets/*.json
data/earth_on_minecraft/settlements/raid_profiles/*.json
```

聚落档案应持久化保存：

- 中心和边界。
- 规模、技术阶段和产业专长。
- 已登记居民和工作岗位。
- 已建功能设施。
- 当前物资需求和供应优势。
- 声望、安全状态和最近灾厄事件。
- 版本与幂等迁移标记。

不能每 tick 扫描整个村庄。聚落状态应以 POI、关键方块变化和低频校验触发更新，并缓存计算结果。

实现前必须从 Minecraft 26.2 和 NeoForge 26.2 实际源码确认村民交易、POI、结构池、战利品和实体数据的注册入口，不沿用旧版本签名猜测。

## 6. 居民角色

### 6.1 保留并增强原版角色

- 农民、渔夫、牧羊人。
- 工具匠、武器匠、盔甲匠、石匠。
- 制图师、图书管理员、牧师和商人。

这些职业的交易应接入本体材料。例如石匠认识水泥、陶瓷和工业骨料；工具匠认识钢材、橡胶密封件和机械零件；制图师可以出售地质或聚落地图。

### 6.2 新增现实角色

第一轮建议覆盖：

- 矿工 / 采石工。
- 地质调查员。
- 机械操作员。
- 机修工 / 技师。
- 电工 / 变电操作员。
- 炉前工 / 冶炼工。
- 化工操作员。
- 水务操作员。
- 仓库管理员。
- 货运调度员。
- 建材工 / 窑工。
- 商店经营者 / 废料回收商。

不通过几十个近似职业堆数量。每个角色必须至少有独立工作站、交易池、服饰特征、工作动作和实际聚落用途。

### 6.3 外观差异

- 在原版村民模型基础上增加职业服饰层、安全帽、护目镜、手套、围裙、反光条和工具。
- 同一角色提供受控的肤色、发色、服装磨损和配件变化。
- 外观不改变碰撞箱，也不把角色做成与 Minecraft 风格割裂的写实人物。
- 儿童继续使用原版年龄系统；老年或资深身份第一阶段作为外观/资历，不另做完整寿命模拟。

### 6.4 其他现实生物

Living World 后续还应覆盖与聚落和自然环境有实际关系的非人类生物，例如传粉昆虫、常见鸟类、区域鱼类、家畜品种、驮运牲畜和城镇周边动物。它们不能只是换皮：至少要与授粉、害虫控制、渔业、畜牧、运输或生态指示中的一种玩法发生关系。

首批先把居民和聚落底座做稳，不同时铺开大量动物，避免再次形成只有模型、掉落和生成蛋而没有用途的内容。

## 7. 交易与物资需求

交易由四层组成：

1. 原版职业基础交易。
2. 居民具体角色交易。
3. 聚落技术阶段和产业专长交易。
4. 本地供应、短缺和玩家声望修正。

交易必须可预测。聚落公告板应显示当前主要收购物资、可供应产品和对应职业，避免玩家逐个点击居民碰运气。

示例：

- 矿业村收购支护木材、照明和食物，出售矿物样品、碎矿和地质线索。
- 机械化集镇收购煤、橡胶、润滑油和钢材，出售管件、轴承、维修件和基础机器零件。
- 电气化工业镇收购铜线、陶瓷绝缘件、燃料和电池材料，出售电缆、开关柜组件、充电电池和自动化部件。
- 水务设施收购石灰、活性炭和过滤材料，出售软化水、污泥处理产品和水务零件。

生成建筑中的机器不能成为免费无限产出。默认采用以下一种或多种约束：

- 设备未通电或缺少关键部件。
- 设备由聚落拥有，只用于视觉和交易模拟。
- 玩家取得使用权需要声望、维修或供货。
- 生成容器只提供少量合理启动物资，不直接赠送高级完整生产线。

## 8. 掉落设计

### 8.1 居民

不鼓励屠杀居民刷资源。普通居民默认不掉落绿宝石和完整机器，只可能掉落少量随身工作物、损坏工具或个人物品；重要随身库存优先由交易、救援或聚落回收处理。

### 8.2 掠夺者与灾厄单位

掠夺者根据目标聚落的技术阶段出现不同角色：

- 普通掠夺者：食物、工具和一般物资。
- 劫掠者：优先搜索仓库和运输节点。
- 破坏者：攻击门、管线、电缆或控制面板。
- 纵火/爆破行为默认不做或严格受配置限制，避免不可恢复的世界破坏。
- 头目或旗手携带地图、失窃物资、损坏零件和聚落线索。

掉落可以包括抢来的物资包、损坏机械零件、地图、工具和少量本体材料；不增加枪械。

掠夺者目标选择、破坏能力和掉落必须数据驱动，并提供“关闭机器破坏”“仅攻击居民”“完整聚落防御”等配置档位。

原版敌对掠夺者没有常规交易，第一阶段不强行让战斗中的敌人打开村民交易界面。若后续需要黑市、投降、赎回或非法货物流通，应新增独立的中立亡命商人/黑市角色，并为其设置清楚的关系和风险规则。

## 9. 功能建筑与城镇生成

### 9.1 现有村庄增强

第一阶段先向原版村庄增加少量可解释建筑：

- 聚落公告板 / 镇务板。
- 地质与矿业小屋。
- 机械维修工坊。
- 仓库或装卸棚。
- 小型水泵房或配电房。

这些建筑必须有对应 POI、居民角色、交易或功能，不能只是装饰空壳。

### 9.2 资源型聚落

- 矿业镇：矿井口、矿石堆场、破碎/筛分设施、机修和货运。
- 采石镇：采石场、骨料区、石工、窑场和建材仓库。
- 林业镇：锯木、木片、纸浆前置和维修设施。
- 河港镇：仓储、泵站、水处理和货运装卸。

### 9.3 工业镇

工业镇按片区生成，而不是一个巨型固定建筑：

- 镇中心和市场。
- 居住区。
- 工坊/轻工业区。
- 工厂或冶炼区。
- 发电站与变电设施。
- 水处理与储水设施。
- 仓库和货运区。

第一版控制规模和人口，避免路径寻找、实体数量、方块实体和粒子造成持续性能压力。

### 9.4 聚落发展

初始版本先在新世界生成固定档案聚落。后续再加入可见的发展过程：

- 玩家供货或修复设施。
- 建造指定基础设施。
- 达到居民、能源和物流条件。
- 由施工框架逐步替换为新建筑。

不能在玩家不知情时大范围改造已有村庄，也不能破坏玩家建筑。旧世界升级必须显式触发、可预览和可取消。

## 10. 聚落界面

新增“聚落公告板/镇务板”界面，使用 Minecraft 原版容器大小和视觉语言，提供：

- 聚落名称、规模、技术阶段和产业专长。
- 人口、职业和空缺岗位。
- 当前短缺、主要收购和主要供应。
- 功能建筑与基础设施状态。
- 当前灾厄风险和最近事件。
- 玩家声望与可解锁服务。
- 指向对应居民、建筑、手册和 JEI 条目的入口。

居民交易界面增加紧凑身份区：姓名、角色、聚落、技能和当前工作，不把完整镇务信息塞进每个交易窗口。

## 11. 高质量动作与机器效果

### 11.1 居民动作

先做可读、短促、与工作职责一致的动作：

- 矿工挥镐、检查岩样、搬运矿袋。
- 机修工使用扳手、检查面板、润滑设备。
- 电工检查绝缘子、开关柜和电缆。
- 炉前工观察炉口、清渣和操作阀门。
- 水务操作员取样、开关泵和检查滤池。
- 仓库人员搬箱、记录货物和装卸输送带。

动作不只是循环摆手。居民应在对应 POI、工作时段和设备状态下执行，并在受惊、交易、休息和灾厄期间正确中断。

### 11.2 机器动作

运行效果必须读取真实 BlockEntity 状态和处理进度：

| 设备 | 主要动作与效果 |
|---|---|
| 颚式破碎机 | 颚板往复、进料抖动、短促粉尘和低频冲击声 |
| 球磨机 | 滚筒连续旋转、启动/停机惯性、轴承声 |
| 筛分机 | 筛面振动、分级物料移动、轻量粉尘 |
| 磁选机 | 磁鼓旋转、精矿流向、受控电磁音效 |
| 浮选槽 | 液位、气泡、泡沫层和搅拌器 |
| 焙烧/还原/窑炉 | 炉温渐变、炉口热光、烟气与渣口状态 |
| 电解槽 | 电极气泡、母排发光、电解液变化 |
| 塔器 | 液位/回流视窗、蒸汽、顶部和底部状态 |
| 输送带 | 皮带运动、可见货物平滑插值、阻塞状态 |
| 汽轮发电机 | 转子启动、稳定转速、负载声音和输出指示 |

多方块形成后应作为一个整体表现：贯通管线、连续壳体、统一运行灯和局部运动部件，而不是每个方块独立随机冒粒子。

### 11.3 声音、粒子和性能

- 每台代表设备需要启动、运行、停机和异常四类反馈，但不要求每类都使用独立长音频。
- 循环声音按距离、遮挡、机器负载和玩家设置衰减。
- 粒子由客户端生成并限制频率；远距离、屏幕外和低画质模式停止高成本效果。
- 提供低/中/高三档机器效果质量。
- 不用粒子数量代替动画质量，不在工业镇让几十台设备持续满负载渲染。

动画技术选型必须先验证 Minecraft 26.2 原版动画能力和可用库的兼容性。原版系统能满足时优先复用；只有复杂实体或多方块确有需要时才引入额外动画依赖。

## 12. 现实基础设施物品

这条主线优先加入能形成实际功能的基础设施，而不是继续增加只用于合成的零件：

- 铁管、钢管和不同口径管道。
- 弯头、三通、法兰、阀门和止回阀。
- 泵、储罐、压力表和流量计。
- 电线杆、路灯、配电箱和小型变电设施。
- 仓储架、装卸口、货箱和输送分流器。
- 道路、排水沟、井盖和管线接口。

钢筋、地基、梁柱受力和结构安全不在本轮实现，避免把公共基础设施路线变成硬核建筑模拟。

## 13. 分阶段实施

### 阶段 0：短稳定化门槛

1. 修正当前多方块控制面板几何与三状态传播。
2. 完成现有 UI、多方块、输送和日志的客户端实机基线。
3. 建立 0.1.9 可复现提交。

这一步只收口已发现问题，不继续扩大工业内容。

### 阶段 1：Living World 基础

1. 建立聚落档案、居民档案、姓名池和角色数据格式。
2. 对现有和新生成村民安全补全稳定姓名、聚落和角色。
3. 新增聚落公告板及第一版界面。
4. 先实现农业村、资源村、机械化集镇三个聚落档案。
5. 先实现矿工、地质调查员、机修工、电工、水务操作员、仓库管理员六个完整角色。

这一阶段不先生成大型城镇，先把身份、查询和数据持久化做稳。

### 阶段 2：交易、掉落和工作行为

1. 原版职业交易接入本体材料。
2. 六个新角色拥有独立交易、POI、服饰和工作动作。
3. 聚落公告板显示需求与供应。
4. 居民和掠夺者掉落改为数据驱动。
5. 旧世界居民迁移、死亡、繁殖、转职和卸载重装测试。

### 阶段 3：原版村庄增强

1. 增加公告板、矿业小屋、维修工坊、仓库、泵房/配电房。
2. 让建筑、职业、交易和聚落档案互相对应。
3. 新世界验证不同生物群系和旋转方向。
4. 兼容其他 mod 对原版村庄结构池和职业的追加。

### 阶段 4：资源镇与工业镇

1. 先实现一个完整矿业镇和一个电气化工业镇。
2. 工业镇包含小型工厂、发电站、变电、水务和仓储片区。
3. 生成设备不免费产出，使用供货、维修、声望和所有权规则。
4. 增加铁/钢管网、阀门、泵、路灯和配电设施。

### 阶段 5：掠夺者与聚落防御

1. 掠夺角色按聚落阶段选择目标和装备。
2. 增加劫掠、破坏和失窃物资逻辑。
3. 默认世界破坏保持可恢复，并允许关闭。
4. 聚落防御奖励接入声望、维修和高级交易，而不是只给一次性战利品。

### 阶段 6：动作、特效和声音全面升级

1. 先完成颚式破碎机、球磨机、浮选槽、还原炉、汽轮发电机五个代表设备。
2. 建立统一的状态动画、声音衰减和效果质量配置。
3. 完成六个首批居民角色的工作动作。
4. 验证性能后扩展到全部机器和城镇设施。

### 阶段 7：发展与生态联动

1. 玩家供货和设施建设驱动聚落可见发展。
2. Earth Human 可选读取居民/聚落环境，但本体不模拟人体难度。
3. 魔法、玄幻和科幻项目只通过稳定材料、聚落和环境接口增加自己的建筑或角色。
4. 发布 `SettlementQueryApi` 只读接口，避免其他项目扫描整个村庄。

## 14. 第一轮生产质量切片

第一轮不是占位 MVP，而是用于确定整个大型系统质量标准的完整纵向切片：

- 三种聚落档案：农业村、矿业村、机械化集镇。
- 六个居民角色及稳定随机姓名。
- 一个聚落公告板界面。
- 六套角色交易和工作动作。
- 五种功能建筑。
- 村民与掠夺者第一版掉落规则。
- 颚式破碎机、球磨机、浮选槽、还原炉、汽轮发电机的运行效果升级。
- 新旧世界、专服、无其他附属、四项目组合和低画质性能验证。

这批通过后再扩大到工业镇和全部机器，避免一次生成几十个同质职业、空壳建筑和只换颜色的动画。

## 15. 验收标准

玩家进入一个聚落后，应能在几分钟内回答：

- 这里叫什么，是村落、集镇还是城镇。
- 主要依靠什么产业，处于什么技术阶段。
- 每个关键居民叫什么、做什么工作、在哪里工作。
- 这里缺什么、收购什么、能提供什么。
- 工厂、发电站和水务设施是否真的有功能。
- 掠夺者为什么攻击这里、抢了什么、如何恢复。
- 机器当前在做什么，为什么有这些动作、声音和产物。

如果只能看到更多村民皮肤、更多房子和更多粒子，但无法回答这些问题，就不算完成这条主线。
